Contributing

Ngx Semantic is a young project. A whole lot remains to be done. Documentation needs to be added and there would be issues that need to be triaged and bug fixes implemented as the library gains traction.

How to start

It is very easy:

  • Find something to do. There's plenty of it, just check the issues page to get ideas.
  • It is suggested to submit an issue first: this can help you avoiding any extra work

Once it is clear what you need to do:

  • Fork the project
  • Clone it:
    $ git clone https://github.com/<your username>/ngx-semantic
  • Make your changes. Here are some good guidelines:
    • Don't worry about making a lot of commits, they will be squashed when the PR is merged
    • Write clear commit messages
    • Respect the coding style. If you don't have it already, install an eslint plugin for your editor
  • Test your code
