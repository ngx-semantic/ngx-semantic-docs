Ngx Semantic

Introduction

Angular and jQuery don't go together - this is the fundamental principle of this library. It provides Angular component versions of the Semantic UI elements, views and modules, so that you don't need to add jQuery to your app.

NOTE: Unlike most other angular libraries that provide Semantic UI support, this library covers much more than the Semantic UI modules. It also provides Angular wrappers for the Elements, Collections, as well as Views.

Installation

First, you'd need to import the Semantic UI CSS styles from a CDN to your index.html file. For best results, make sure you are referencing the latest Semantic UI version.

Next, you'd need to install the NgxSemantic package as well as the Angular CDK from the npm registry

or if you're into yarn

Usage

NgxSemantic allows for individual module imports which allows for Ivy to shake off unused modules in prod builds leading to smaller bundle sizes and faster load times. But, if you want to import all the goodness that NgxSemantic offers, then include the root module in your app.module.ts file

Note that you can also import the modules individually.

You're good to go.

Dependencies

Note that you can also import the modules individually.

  • Angular (^9.0.0)
  • Semantic UI CSS (^4.2.1) (Remember jQuery and the Semantic UI JS files are not required or used)
